# 정보관 1층~9층 강의실 예약 시스템 요구사항 작성안

작성일: 2026-05-09

## 작성 원칙

원본 요구사항은 삭제하지 않고 유지하되, 운영 담당 역할은 `관리자`로 통합했다. 시스템 사용자는 `학생`, `교수`, `관리자` 3개 역할만 가진다.

## 기능 묶음

1. 계정/권한
2. 강의실/시설 관리
3. 시간표/현황 조회
4. 예약 신청/시설 확인
5. 승인/대기/취소
6. 알림/사용 확인
7. 백업/복구/운영

## 매력적인 요구사항

| 기능 | 설명 |
|---|---|
| 층별 대시보드 | 1층~9층 예약률과 빈 강의실 수를 색상으로 표시 |
| 시설 필터 | 빔프로젝터, PC, 전자칠판, 콘센트, 접근성 기준으로 검색 |
| 충전 가능 좌석 안내 | 콘센트 수, 콘센트 위치 설명, 충전 가능 좌석, 충전기 대여 가능 여부 표시 |
| 시설 불편 신고 | 콘센트 고장, PC 문제, 빔프로젝터 문제, 청결 문제 신고 및 관리자 처리 |
| 체크인/노쇼 방지 | 예약 시작 전후 사용 확인으로 빈 공간 회수 |

## 결론

이 프로젝트는 서버 하나 안에서 기능별 패키지를 명확히 나누는 모듈러 모놀리스가 적합하다. 정보관 1층~9층 확장, 강의실 시설 검색, 콘센트/충전 가능 좌석 안내, 시설 불편 신고, 체크인을 추가하면 설계지침서의 “매력적인 요구사항”에도 잘 맞는다.
