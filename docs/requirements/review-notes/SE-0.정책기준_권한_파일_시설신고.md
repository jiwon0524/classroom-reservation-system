## 층별 혼잡도/공간 사용률

```text
층별 혼잡도(%) =
  (정규 강의 시간 + 승인 예약 시간 + 사용 불가 시간)
  / (활성 강의실 수 * 운영 시간)
  * 100
```

기본 운영 시간은 `09:00~22:00`이다.

| 혼잡도 | 표시 |
|---:|---|
| 0~39% | 여유 |
| 40~69% | 보통 |
| 70~89% | 혼잡 |
| 90~100% | 매우 혼잡 |

강의실별 사용률은 선택 기간의 승인 예약 시간과 정규 강의 시간을 합산한 뒤 해당 강의실의 전체 운영 가능 시간으로 나누어 계산한다.

## 파일 저장

저장 기준:
- temp 파일에 먼저 저장한다.
- 스키마 검증에 성공하면 원본 파일을 교체한다.
- 저장 실패 시 원본 파일은 유지한다.
- 예약 생성/승인/취소는 `roomId + date` 단위 잠금을 건다.

## 예약 변경/대기 승격

예약 변경 요청은 예약 시작 1시간 전까지만 허용한다. 변경 승인 전에는 기존 예약을 유지하고, 관리자가 승인할 때 시간 충돌, 역할별 최대 예약 시간, 학생 정원 50% 규칙을 다시 검사한다.

대기 예약은 취소, 노쇼 취소, 변경 승인으로 빈 시간이 생겼을 때 같은 강의실/시간대 대기열에서 신청 시각이 빠른 순서로 자동 승격한다. 승격 결과는 알림과 감사 로그에 남긴다.

## 사용 후 점검/운영 공지

예약 종료 후 24시간 이내에는 사용자가 이상 없음 또는 시설 문제를 기록할 수 있다. 시설 문제를 선택하면 시설 신고가 `RECEIVED` 상태로 생성된다.

운영 공지는 `INFO`, `WARNING`, `BLOCKING` 3단계로 관리한다. `BLOCKING` 공지가 적용된 강의실과 시간대에는 신규 예약을 차단하고 기존 예약은 관리자 검토 대상으로 표시한다.

## 시설 신고 상태

상태는 네 개만 둔다.

```text
RECEIVED -> IN_PROGRESS -> RESOLVED
RECEIVED -> REJECTED
IN_PROGRESS -> REJECTED
REJECTED -> RECEIVED
```

`RESOLVED`는 종료 상태로 둔다.
