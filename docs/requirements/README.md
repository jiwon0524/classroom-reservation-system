# 요구사항 문서 정리

## 제출/확인용

- `SE-0.SW요구사항-강의실예약시스템.xlsx`

위 엑셀 하나가 요구사항 통합본입니다. StRS, SRS, 추적표, 권한표, 정책기준, 파일저장기준, SOLID 검토, 설계지침 반영 내용을 시트로 모았습니다.

## 작업 원본

- `source-csv/`

엑셀을 만들 때 사용한 CSV 원본입니다. 요구사항을 수정할 때는 이 CSV를 고친 뒤 엑셀을 다시 만드는 방식으로 관리하면 됩니다.

## 검토 메모

- `review-notes/`

역할 3개 정리, 6주 4인 범위 평가, 정책 기준, 핵심 기능 점검 같은 검토 메모입니다. 제출용이라기보다는 발표 준비와 구현 범위 조절용입니다.
